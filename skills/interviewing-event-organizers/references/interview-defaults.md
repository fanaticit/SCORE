# Interview Defaults

Use these defaults to reduce unnecessary questions during organizer interviews. Present
defaults as recommendations and let the organizer override them.

## Activity Defaults

| Activity | Default to suggest | Why | Key follow-up questions |
| --- | --- | --- | --- |
| Chess | Swiss | Most chess tournaments use Swiss when many players should keep playing and receive a fair overall ranking. | Round count, round duration, tie-breaks, online or in-person, individual or team event. |
| Volleyball | No draws | Volleyball normally has decisive sets or match winners. | Sets or points target, win-by-two rule, courts, rotation/substitution rules. |
| Football | Draws allowed | Football league/group matches often allow draws unless the event needs immediate elimination. | Group vs knockout, points for win/draw, penalty shootout rules, pitch count. |
| Basketball | No draws by default | Basketball usually requires a winner; overtime or sudden-death rules may be needed in finals. | Match length, overtime rule, courts, team count. |
| Netball | No draws by default for finals; draws can be allowed in league stages if organizer wants. | Netball events often need clear finalists but may allow tied pool games. | Match length, court count, points for draw, final tie-break. |

## Chess Format Prompt

When chess is included, say:

"Swiss is the usual default for chess because it gives structured pairings and a fairer
overall ranking while keeping players involved. You can also choose Knockout for maximum
drama, or Arena for flexible, fast-paced online play."

Then offer:

- Swiss: best for balanced competition and full-event experience.
- Knockout: best for drama, match pressure, and clear brackets.
- Arena: best for convenience, volume, and quick online action.

## Question Reduction Rules

- If chess is requested and no format is specified, recommend Swiss and ask whether to
  confirm Swiss or compare alternatives.
- If volleyball is requested, assume no draws unless the organizer explicitly says otherwise.
- If football is requested, allow draws in group/league stages unless the organizer wants
  every match to produce a winner.
- Ask for resources only once: courts, fields, tables, rooms, or online platform.
- Ask for privacy preferences once, but always flag contact fields for privacy review.
- Ask whether players or teams can be added after the tournament starts. Default to yes for
  community events, but explain that the site then needs late-entry admin controls.
- At the end of the interview, ask: "Are there any extra requests or event-specific features
  you want included before I create the site brief?"

## Tournament Operation Defaults

- Late entrants: allowed by default. Admins should be able to add a player or team after the
  event starts, then place them into future fixtures without rewriting completed results.
- First-round pairing: allow some randomness or seeding control instead of sorting only by
  names. The organizer should be able to reset and regenerate the first round if pairings do
  not work.
- Later Swiss rounds: pair from standings/ranking while avoiding repeat pairings where
  practical.
- Extra games: if Swiss finishes too quickly, offer an optional round-robin, extra friendly
  rounds, or a final so entrants can keep playing.
